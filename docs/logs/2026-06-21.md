# 2026-06-21 작업 로그

## Testcontainers 통합 테스트 도입

- `auth-service` 테스트 의존성에 Spring Boot Testcontainers와 Testcontainers JUnit Jupiter 모듈을 추가했다.
- Spring Boot 3.3.6 기본 Testcontainers 1.19.8 대신 `testcontainers.version=1.21.3`을 사용하도록 조정했다.
- PostgreSQL, MongoDB, Elasticsearch, Redis, Kafka를 테스트 실행 시 실제 컨테이너로 띄우는 통합 테스트를 추가했다.
- 테스트는 JPA 사용자 저장, MongoDB 메시지 저장, Elasticsearch 검색 문서 저장, Redis key/value 저장, Kafka 메시지 발행을 검증한다.
- 외부 Eureka, OpenAI, Tempo, S3 의존성은 테스트에서 사용하지 않도록 속성을 고정했다.
- Docker Desktop 29에서 docker-java 기본 API 버전 `1.32`로 `/info`를 호출하면 빈 Docker 정보가 반환되어 Testcontainers가 Docker를 감지하지 못했다. 테스트 JVM에 `api.version=1.54`를 명시해 현재 Docker daemon API와 맞췄다.
- 로컬 macOS에서는 실제 Docker Desktop socket인 `~/.docker/run/docker.sock`을 우선 사용하고, Jenkins/Linux 환경에서는 `/var/run/docker.sock`을 fallback으로 사용한다.
- 검증 명령 `./gradlew :auth-service:cleanTest :auth-service:test --tests com.kafka.auth.infra.InfrastructureIntegrationTest --info`를 실행했고, 결과는 `tests=1, skipped=0, failures=0, errors=0`이다.

## 적용 개념

- Testcontainers: 테스트 코드가 Docker 컨테이너를 직접 띄워 실제 인프라와 거의 같은 조건에서 통합 테스트를 수행하는 도구다.
- DynamicPropertySource: 컨테이너가 실행된 뒤 할당된 포트와 접속 URL을 Spring Boot 테스트 환경 속성으로 주입한다.
- disabledWithoutDocker: Docker가 없는 환경에서는 통합 테스트를 실패시키지 않고 비활성화한다. Jenkins 로컬 CD 환경처럼 Docker가 있는 곳에서는 실제로 실행된다.
- docker-java API version: Testcontainers가 Docker daemon과 통신할 때 사용하는 REST API 버전이다. Docker Desktop이 너무 낮은 API 버전에 대해 호환 응답을 제대로 주지 않으면 `api.version`을 현재 daemon API에 맞춰 고정해야 한다.
