# 2026-07-04 작업 로그

## 작업

- 이메일 인증 메일 제목이 일부 메일 클라이언트에서 깨지는 문제를 재현 가능한 단위 테스트로 고정
- `MimeUtility.encodeText(..., "UTF-8", "B")`로 제목 헤더를 명시 인코딩하도록 메일 발송 로직 보강
- 이메일 인증 메일 본문을 HTML 템플릿으로 정리하고, 텍스트 대체 본문을 함께 제공
- 로컬 `.env`를 Spring Boot가 직접 읽을 수 있도록 `spring.config.import` 경로를 추가
- 로컬 Kubernetes에서 Elasticsearch, Logstash, discovery-service가 메모리 압박으로 흔들리지 않도록 JVM heap, probes, resources를 보강
- k6 기반 부하 테스트 시나리오를 추가해 로그인, 채팅방, 메시지 전송, 검색/자동완성 API를 smoke/load/stress 프로파일로 검증할 수 있게 구성

## 개념

- MIME encoded-word: 메일 제목처럼 헤더 영역에 한글이 들어갈 때는 본문과 별도로 RFC 2047 형식의 인코딩이 필요하다. 메일 클라이언트가 헤더 charset을 잘못 추정하면 제목만 깨질 수 있으므로, 제목 헤더를 `=?UTF-8?B?...?=` 형태로 명시 인코딩한다.
- HTML 메일과 plain text 대체 본문: HTML을 지원하지 않는 클라이언트나 보안 정책으로 HTML이 차단되는 환경을 위해 텍스트 본문도 함께 넣어야 한다.
- Kubernetes resources/probes: 로컬 Docker Desktop Kubernetes는 단일 노드라 ELK 계열 JVM 프로세스가 쉽게 메모리 경쟁을 일으킨다. heap과 resources를 같이 제한하고 readiness/liveness probe를 두면 시작 중인 서비스로 트래픽이 들어가거나 OOM 후 방치되는 상황을 줄일 수 있다.
- k6 부하 테스트: 기능 테스트가 “정상 동작 여부”를 본다면, k6는 일정 동시 사용자 수에서 응답 시간, 실패율, 병목 지점을 확인한다. Kafka Talk에서는 메시지 전송 후 Kafka, MongoDB, Elasticsearch, Redis까지 이어지는 비동기 흐름이 있어 API 응답 시간과 백그라운드 처리 지표를 함께 보는 것이 중요하다.
