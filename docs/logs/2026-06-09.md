## 사용한 개념

- JWT: 로그인 성공 후 서버가 서명한 토큰을 발급하고, 이후 요청에서 `Authorization: Bearer` 헤더로 인증한다.
- Spring Security Filter: 요청마다 JWT를 검증하고 SecurityContext에 사용자 인증 정보를 넣는다.
- BCrypt: 일반 로그인 비밀번호를 단방향 해시로 저장한다.
- Email Verification Code: 개발 단계에서는 인증코드를 서버 로그와 응답에 노출하고, 운영 단계에서는 메일 발송 서비스로 교체한다.
- Eureka: 여러 Spring 서비스가 서로를 찾을 수 있도록 서비스 디스커버리 서버를 둔다.
- Vite Proxy: React 개발 서버에서 `/api` 요청을 백엔드 `localhost:8890`으로 프록시한다.
- Gradle Multi-project Build: 여러 Spring Boot 서비스를 하나의 Gradle 루트에서 관리한다.
- Docker Multi-stage Build: Gradle/Node 빌드 단계와 경량 런타임 이미지를 분리한다.
- Kubernetes Deployment/Service: 각 서비스를 파드로 배포하고 NodePort/ClusterIP로 접근 경로를 나눈다.
- Motion: React 컴포넌트의 마운트/전환 애니메이션을 선언적으로 처리한다.
- PostgreSQL: 유저, 채팅방, 권한, 멤버십처럼 정합성과 관계가 중요한 데이터를 저장한다.
- MongoDB: 대량으로 계속 추가되는 채팅 메시지 원본을 문서 형태로 저장한다.
- Elasticsearch: 채팅방 이름, 설명, 메시지 본문 검색을 빠르게 처리하는 검색 엔진이며 원본 DB가 아니라 검색 인덱스로 사용한다.
- Kafka: 메시지 발송 이벤트를 producer와 consumer 사이에서 비동기 스트림으로 전달한다.
- Kafka advertised listeners: Docker 네트워크 안에서는 `kafka:9092`, 로컬 개발 서버에서는 `localhost:29092`처럼 클라이언트 위치에 따라 접근 가능한 주소를 다르게 알려줘야 한다.
- WebSocket/STOMP: HTTP 요청/응답을 넘어서 브라우저와 서버가 실시간 양방향 메시지를 주고받게 한다.
- Jenkins Pipeline: GitHub 소스를 받아 테스트, 빌드, 이미지 생성, Kubernetes 반영 같은 반복 배포 단계를 선언형 파이프라인으로 실행한다.
- Docker socket mount: Jenkins 컨테이너가 호스트 Docker에 명령을 내릴 수 있게 하지만 권한이 강하므로 로컬 배포 전용으로 분리한다.
- kubeconfig mount: Jenkins가 로컬 Kubernetes cluster에 `kubectl` 명령을 실행할 수 있게 하는 인증/접속 설정이다.

## Docker와 Kubernetes를 같이 쓰는 이유

- Docker는 애플리케이션 실행 환경을 이미지로 포장하는 기술이다. 백엔드는 Java 21과 Spring Boot 실행 환경, 프론트엔드는 React 빌드 결과와 Nginx 실행 환경을 이미지에 담는다.
- Docker 이미지를 사용하면 로컬 PC, 다른 개발자 PC, 서버, Kubernetes 환경에서 같은 방식으로 실행할 수 있다.
- Kubernetes는 Docker 이미지로 만들어진 컨테이너들을 배포하고 연결하고 관리하는 기술이다.
- 이 프로젝트에서는 `auth-service`, `discovery-service`, `frontend`, `kafka`를 각각 컨테이너 이미지 또는 외부 이미지로 준비하고 Kubernetes Deployment와 Service로 연결한다.
- 프론트엔드는 `/api` 요청을 `auth-service:8890`으로 보내고, 백엔드는 `discovery-service:8761/eureka`에 서비스 정보를 등록한다.
- Docker는 실행 단위를 포장하고, Kubernetes는 그 실행 단위를 여러 서비스로 배치해 운영하는 역할을 맡는다.

## 로컬 Kubernetes 사용 방법

1. Docker Desktop에서 `Settings -> Kubernetes -> Enable Kubernetes`를 켠다.
2. 현재 kubectl context를 확인한다.

```bash
kubectl config current-context
```

3. 로컬 이미지를 빌드한다.

```bash
scripts/docker-build.sh
```

4. Kubernetes 매니페스트를 적용한다.

```bash
scripts/k8s-apply.sh
```

5. 브라우저에서 접속한다.

```text
프론트엔드: http://localhost:30880
백엔드: http://localhost:30890
```

## 추가로 사용한 개념

- Direct Message Room: 사용자 두 명의 이메일을 정렬해서 `directKey`를 만들고 같은 두 사용자 조합에는 하나의 방만 생성한다.
