## 사용한 개념

- JWT: 로그인 성공 후 서버가 서명한 토큰을 발급하고, 이후 요청에서 `Authorization: Bearer` 헤더로 인증한다.
- Spring Security Filter: 요청마다 JWT를 검증하고 SecurityContext에 사용자 인증 정보를 넣는다.
- BCrypt: 일반 로그인 비밀번호를 단방향 해시로 저장한다.
- Email Verification Code: 개발 단계에서는 인증코드를 서버 로그와 응답에 노출하고, 운영 단계에서는 메일 발송 서비스로 교체한다.
- Eureka: 여러 Spring 서비스가 서로를 찾을 수 있도록 서비스 디스커버리 서버를 둔다.
- Vite Proxy: React 개발 서버에서 `/api` 요청을 백엔드 `localhost:8890`으로 프록시한다.
- Gradle Multi-project Build: 여러 Spring Boot 서비스를 하나의 Gradle 루트에서 관리한다.
- Docker Multi-stage Build: Gradle/Node 빌드 단계와 경량 런타임 이미지를 분리한다.
- Kubernetes Deployment/Service: 각 서비스를 파드로 배포하고 NodePort/ClusterIP로 접근 경로를 나눈다.
- Motion: React 컴포넌트의 마운트/전환 애니메이션을 선언적으로 처리한다.
