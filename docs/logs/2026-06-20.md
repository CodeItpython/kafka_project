# 2026-06-20 작업 로그

## Elasticsearch 검색 고도화 1차

- `chat-messages` 인덱스의 `content`, `roomName`, `senderName` 필드를 `search_as_you_type` 매핑으로 변경했다.
- 검색 쿼리를 Spring Data repository derived query에서 Elasticsearch native query 기반 `bool_prefix` 검색으로 분리했다.
- `content._2gram`, `content._3gram`, `content._index_prefix` 같은 보조 필드를 함께 조회해 사용자가 검색어를 입력하는 중에도 후보가 더 잘 나오도록 했다.
- 기존 인덱스가 옛 `text` 매핑이면 시작 시 인덱스를 재생성하고 MongoDB의 `chat_messages` 원본을 Elasticsearch에 다시 색인하도록 구성했다.
- Elasticsearch 장애 또는 쿼리 실패 시 기존 repository 검색으로 fallback한다.

## 적용 개념

- `search_as_you_type`: 자동완성/부분 입력 검색에 맞춘 Elasticsearch 필드 타입이다. 입력 prefix를 빠르게 찾기 위해 shingle과 index prefix 보조 필드를 생성한다.
- Read Model: MongoDB가 메시지 원본이고 Elasticsearch는 검색용 읽기 모델이다. 검색 인덱스는 손실되어도 원본 DB에서 다시 만들 수 있다.
- Reindex: 매핑 변경처럼 기존 인덱스를 그대로 수정할 수 없는 경우, 인덱스를 다시 만들고 원본 데이터를 새 매핑으로 다시 넣는 작업이다.
- Fallback Search: Elasticsearch가 일시적으로 실패해도 MongoDB 기반 검색으로 최소 기능을 유지하는 방식이다.
