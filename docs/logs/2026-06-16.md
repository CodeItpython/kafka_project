# 2026-06-16 작업 로그

## Jenkins 로컬 CD 안정화

- `kafka-chat-main-ci`는 main 브랜치 기준 검증용 CI로 유지하고, `kafka-chat-local-deploy`는 CI 성공 후에만 실행되도록 정리했다.
- 기존 CD는 `pollSCM`과 upstream 트리거가 동시에 걸려 같은 커밋에서 배포가 중복 실행될 수 있었다.
- Docker 이미지 빌드 중 `repo.maven.apache.org: Name does not resolve` 오류가 발생해 auth-service 이미지 생성이 실패했다.
- 원인은 Docker 빌드 컨테이너 내부에서 Gradle이 Maven Central 의존성을 다시 받는 구조였기 때문이다.
- Jenkins 워크스페이스에서 `bootJar`와 `npm run build` 산출물을 먼저 만들고, Docker는 산출물만 담는 `Dockerfile.runtime` 경로를 추가했다.

## 적용 개념

- CI: 코드 검증 단계. 백엔드 테스트, 프론트엔드 빌드가 성공하는지 확인한다.
- CD: 검증된 산출물을 Docker 이미지로 만들고 Kubernetes Deployment 이미지 태그를 교체한다.
- Runtime Image: 소스와 빌드 도구를 이미지 안에 넣지 않고, 실행에 필요한 JAR 또는 정적 파일만 담는 이미지다.
- Upstream Trigger: `kafka-chat-main-ci` 성공 결과를 기준으로 `kafka-chat-local-deploy`를 자동 실행한다.
