# 2026-06-28 작업 로그

## 채팅 읽음 상태

- PostgreSQL `chat_room_read_states` 테이블로 채팅방별 사용자 마지막 읽음 시각을 저장하도록 추가했다.
- 메시지 응답에 `readCount`, `deliveryStatus`를 포함해 내가 보낸 메시지가 상대에게 읽혔는지 프론트에서 표시할 수 있게 했다.
- 메시지 목록 조회와 명시적 읽음 처리 시 Redis unread count를 삭제하고, WebSocket `/topic/rooms/{roomId}/read-receipts`로 읽음 상태를 broadcast하도록 구성했다.
- React 화면에 내 메시지의 `전송됨`, `읽음 N` 표시를 추가했고, 상대가 채팅방을 열거나 메시지를 읽으면 실시간으로 갱신되도록 했다.

## 사용한 개념

- 읽음 상태 영속화: 사용자별 마지막 읽음 시각은 휘발되면 안 되므로 PostgreSQL에 저장한다.
- Redis 상태 관리: 채팅방 목록의 unread count는 빠르게 바뀌는 화면 상태라 Redis key 삭제/증가로 관리한다.
- WebSocket fan-out: 읽음 처리 결과를 같은 채팅방 구독자에게 즉시 전파해 화면 상태를 갱신한다.
- Testcontainers 검증: PostgreSQL, Redis 기반 읽음 상태와 unread 초기화 흐름을 통합 테스트에 포함한다.
