# 2026-06-11 작업 로그

## 작업 내용

- 사용자가 Jenkins 로컬 배포 모드의 위험을 이해하고 승인했다.
- `docker-compose.jenkins-deploy.yml` 오버라이드로 Jenkins를 재기동했다.
- Jenkins 컨테이너에 Docker socket이 `/var/run/docker.sock`으로 mount된 것을 확인했다.
- Jenkins 컨테이너에 `~/.kube`가 `/var/jenkins_home/.kube`로 read-only mount된 것을 확인했다.
- Jenkins 내부에서 `docker ps`가 동작하는 것을 확인했다.
- `kafka-chat-ci`와 `kafka-chat-local-deploy` job 모두 `H/1 * * * *` Poll SCM 트리거를 유지하고 있는 것을 확인했다.

## 확인된 이슈

- 로컬 `kubectl config get-contexts` 결과가 비어 있었다.
- Jenkins 컨테이너 내부에서도 `kubectl config current-context`가 설정되어 있지 않았다.
- 따라서 현재 상태에서는 Jenkins가 Docker 이미지를 빌드할 수는 있지만, 로컬 Kubernetes 배포 단계는 Docker Desktop Kubernetes context를 먼저 설정해야 성공한다.

## 다음 조치

- Docker Desktop 설정에서 Kubernetes를 활성화한다.
- 터미널에서 `kubectl config get-contexts`와 `kubectl config current-context`가 정상 출력되는지 확인한다.
- context가 정상화되면 `kafka-chat-local-deploy` job을 실행해 K8s 배포까지 검증한다.
