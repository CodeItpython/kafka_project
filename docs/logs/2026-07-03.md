## 개념

- SMTP: 백엔드가 메일 서버에 접속해 이메일을 발송하는 표준 프로토콜이다. Gmail은 계정 비밀번호가 아니라 앱 비밀번호를 사용한다.
- 인증코드 해시 저장: 4자리 원문 코드는 메일로만 보내고 DB에는 이메일과 코드 조합의 SHA-256 해시를 저장해 로그/DB 노출 위험을 줄인다.
- 최신 코드만 유효: 새 인증코드를 요청하면 기존 미사용 코드를 폐기해 여러 코드가 동시에 살아 있는 상황을 막는다.
- 이메일 인증 rate limit: IP 단위 제한만으로는 특정 이메일 주소에 대한 메일 폭탄을 막기 어렵다. Redis에 `email-verification:send-cooldown:{email}`과 `email-verification:verify-failures:{email}` 키를 두어 이메일 주소별 재발송 간격과 코드 검증 실패 횟수를 제한한다.
- Testcontainers wait strategy: 컨테이너 시작 여부를 특정 로그 문구에만 의존하면 이미지 버전이나 로그 포맷 변화에 취약하다. MongoDB 통합 테스트는 포트 listen 기준 대기로 보강해 CI flake 가능성을 낮췄다.
- Kubernetes resources: requests는 스케줄러가 필요한 최소 자원을 고려하게 하고, limits는 컨테이너가 사용할 수 있는 최대 자원을 제한한다. JVM 서비스는 heap을 명시하지 않으면 컨테이너 메모리와 노드 압박에 따라 OOM이 발생하기 쉬워 로컬 k8s에서도 heap과 probes를 함께 잡는 것이 안정적이다.
