## 그룹방 참여자 초대/나가기

- 새 그룹 채팅방 생성 시 만든 사람을 `chat_room_participants`에 자동 등록하도록 변경했다.
- `GET /api/chat/rooms/{roomId}/participants` API로 채팅방 참여자와 온라인 상태를 조회할 수 있게 했다.
- `POST /api/chat/rooms/{roomId}/participants` API로 그룹방 참여자가 친구를 초대할 수 있게 했다.
- `DELETE /api/chat/rooms/{roomId}/participants/me` API로 그룹방에서 나갈 수 있게 했다.
- 프론트엔드 채팅창 상단 아래에 참여자 패널을 추가하고, 그룹방에서는 친구 초대 select와 나가기 버튼을 표시한다.
- 기존에 참여자 목록이 비어 있던 그룹방은 호환을 위해 공개방처럼 처리하고, 새 그룹방부터 명시 참여자 기반으로 동작하게 했다.
- main에 이미 반영된 로컬/원격 브랜치와 현재 개발 브랜치의 차이를 확인했다.
- 검증 명령 `./gradlew :auth-service:test`와 `npm run build`를 실행했고 둘 다 성공했다.

## 사용한 개념

- Explicit room membership: 채팅방 접근, presence, unread, 알림 대상을 `chat_room_participants` 기준으로 계산한다.
- Backward compatibility: 기존 데이터 중 참여자 목록이 비어 있는 그룹방은 전체 공개방으로 해석해 갑작스러운 접근 차단을 피한다.
- Progressive enhancement: 방 목록에 `participantCount`를 포함해 프론트엔드가 별도 조회 없이 목록에서 참여자 수를 표시할 수 있게 한다.

## 브라우저 알림 표시 개선

- STOMP 실시간 알림을 수신했을 때 앱 내부 알림 목록만 갱신하던 흐름에 브라우저 Notification API 표시를 추가했다.
- Firebase/FCM 환경변수가 없어도 로컬 개발에서는 로그인 후 알림 권한만 허용하면 새 메시지 알림이 OS 브라우저 알림으로 표시된다.
- FCM 설정이 있는 경우에는 기존 서비스워커 기반 백그라운드 푸시 토큰 등록을 유지한다.
- 백그라운드 푸시 알림을 클릭해 `/?roomId=...`로 앱이 열리면 해당 채팅방을 자동으로 열도록 딥링크 처리를 추가했다.

## 사용한 개념

- Notification API: 브라우저가 제공하는 로컬 알림 기능으로, 권한이 `granted`일 때 `new Notification(...)`을 호출해 OS 알림을 띄운다.
- Progressive fallback: FCM 설정이 없거나 FCM 등록에 실패해도 WebSocket/STOMP 실시간 이벤트를 받은 열린 브라우저 탭에서는 알림을 표시한다.
- Notification deep link: 알림 클릭 시 대상 채팅방 ID를 URL 쿼리와 앱 상태로 연결해 바로 대화방으로 이동한다.
